# xray-system-vpn (Ubuntu 22.04+)

Системный VPN-клиент на базе Xray, который запускается прямо из папки проекта (без установки в `/usr/local`).

Команды:
- `./vpn` (одна команда: проверки + меню)
- `./vpn-on`
- `./vpn-off`
- `./vpn-status`
- `./vpn-menu`

Для правильной отработки TUN может понадобиться запуск из под root.

или напрямую:
- `sudo ./vpnctl.py` (по умолчанию: menu)
- `sudo ./vpnctl.py on|off|status|menu`

## Что делает

- автоматически определяет активный uplink (`wlan0`, `eth0`, `tailscale0`, ...);
- поднимает Xray и ждёт интерфейс `xray0`;
- переключает маршрутизацию в `full` или `split` режиме;
- сохраняет доступ к локальным маршрутам через `ip rule ... suppress_prefixlength 0`;
- проверяет при старте CLI:
  - расположение рабочих директорий проекта,
  - список сетевых интерфейсов,
  - наличие профилей,
  - наличие `xray` для команд запуска;
- при запуске меню показывает визуальный bootstrap-экран проверок и список интерфейсов;
- если `xray` не найден, предлагает автоматическую установку;
- после проверок ожидает Enter и открывает главное меню;
- перед каждым `on` делает обязательные бэкапы:
  - сетевых таблиц/правил,
  - DNS-конфигов,
  - локальных конфигов приложения;
- автоматически чистит старые бэкапы и хранит только 4 последних запуска;
- добавляет bypass-правила для DNS-серверов и IP VPN-сервера (чтобы не терялся интернет из-за route-loop/резолва);
- проверяет, что VPN-сервер и DNS после переключения маршрутов не ушли в `xray0`, иначе выполняет rollback;
- при `off` восстанавливает DNS из сохранённого бэкапа и очищает policy routing;
- при ошибке запуска делает безопасный откат.
- отображает в меню шапку со статусом VPN, активным профилем, состоянием профиля и тестом подключения;
- при запуске выполняет полную проверку соединения и сохраняет её результат для главного меню;
- после `VPN ON`/`VPN OFF` повторно выполняет проверку и сразу обновляет результат в шапке меню;
- ручная проверка через пункт статуса также обновляет общий результат на главном экране;
- поддерживает переключение языка интерфейса RU/EN.

## Структура данных (внутри проекта)

- профили Xray: `./data/profiles`
- настройки клиента: `./data/client-settings.json`
- бэкапы запусков: `./backups/<YYYYMMDD-HHMMSS>`
- runtime/log/state: `/tmp/xray-system-vpn`

Важно: все импортируемые через CLI Xray-конфиги сохраняются в `./data/profiles`.

## Быстрый старт
0. Делаем файлы исполняемыми и запускаем меню:
   
   ```bash
   chmod +x vpnctl.py && chmod +x vpn
   ```
   ```bash
   sudo ./vpn
   ```

2. Импорт профиля из ссылки:

```bash
sudo ./vpnctl.py import-link 'vless://UUID@host:443?...#MyProfile' --activate --name my-vless
```

или VMess:

```bash
sudo ./vpnctl.py import-link 'vmess://BASE64_JSON' --activate --name my-vmess
```

2. Включить VPN:

```bash
./vpn-on
```

или открыть меню одной командой:

```bash
./vpn
```

В меню:
- пункт `5` — выбор активного профиля по номеру из пронумерованного списка;
- пункт `6` — переключение языка интерфейса;
- во всех подменю `0` — назад/отмена, в главном меню `0` — выход.

3. Проверить статус:

```bash
./vpn-status
```

4. Выключить VPN:

```bash
./vpn-off
```

## Профили и настройки

Список профилей:

```bash
sudo ./vpnctl.py list
```

Выбрать активный профиль:

```bash
sudo ./vpnctl.py use my-vless
```

Режимы маршрутизации:

```bash
sudo ./vpnctl.py config --route-mode full
sudo ./vpnctl.py config --route-mode split --split-interfaces wlan0,eth0
sudo ./vpnctl.py config --language ru
sudo ./vpnctl.py config --language en
```

## DNS-бэкапы

Перед каждым `sudo ./vpnctl.py on` создаются DNS-бэкапы:
- рядом с оригиналами (`/etc/resolv.conf.xsysvpn-...`, `/etc/systemd/resolved.conf.xsysvpn-...`);
- и копия в `./backups/<timestamp>/dns/`.
- сохраняются только 4 последних комплекта бэкапов.
